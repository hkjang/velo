# `was-bootstrap` / `was-admin` 모듈 개요

이 두 가지 모듈은 Velo WAS 생태계의 운영 및 실행 구동 영역을 담당하는 상단 프론트엔드 모듈 계층입니다.

## 1. `was-bootstrap` (구동 시작 엔진)
`was-bootstrap` 모듈은 사실상 애플리케이션의 `main()` 구동계입니다. 애플리케이션 환경을 작동 시 위해 분리된 모든 하위 핵심 라이브러리(네트워크, 파서, 프로토콜 구조, 환경설정, 배포계) 를 모두 통합 접합하여 애플리케이션 초기 서버 리스너 구조를 결집시킵니다.

- 의존된 구성요소 로합(`SnakeYAML` 엔진을 통한 파일 로딩), 프로토콜 어댑터 준비 과정, Netty 의 서버 부트스트랩핑 구조 확립 및 내부 내장된 서블릿(Proxy) 환경 로드를 수행합니다.
- `ServletContainer` 에게 최종 요청을 위임하기 전 앞서 모듈에 얽혀야할 기본 상태관찰자용 헬스포인트(`health`,`info`) 나 관리를 위한 동적 환경을 사전 등록시킵니다.
- 실제 배포판 없이 서버의 시작 종단간 환경을 모의로 점검하는 가상 Mock 배포(예: `LiveWarDeploymentTest`)나 가벼운 내장 테스트 유닛 환경을 포괄합니다.

## 2. `was-admin` (관리 CLI 모듈 및 자동화 시스템)
안정적이고 강력하며 확고한 콘솔 운영 체계를 수립하기 위해, 본래 JEUS 의 구조 명령기인 `jeusadmin` 과 완벽히 유사하게 구현된 CLI 관리툴 역할을 담당합니다. 

- **조작 터미널**: 전통적인 터미널 환경 감별에 익숙치않은 작업자를 배려해 자동완성(Auto Completions) 기능이나 앞서 입력된 조견 히스토리 조작계를 담보하는 자바 라이브러리 `JLine` 을 기반으로 구성되게 설계되었습니다.
- **풍부한 기능 명령어 조치**: 환경 구역 설정 도메인부터, JVM 메모리 체크, 동시 접속 현존 스레드 현황, 라이브 프로파일 체크링, 활력 로깅레벨 단계 실시간 재정립, MBean JMX 호출 등 실로 14가지 영역 73여가지 체계적인 명령 프롬프트들을 제공합니다.
- **수행 환경**: 시스템 명령 요청 호출시 각 API 처리 접합부, 백엔드 명령(`AdminClient`) 구조인 `LocalAdminClient`(가상메모리 내부상황 모니터링) 또는 API를 통합하기 위한 확장 규격인 `RemoteAdminClient`를 향토적으로 지목하며 수행 환경체제를 전달합니다.
- **반복 자동화 제어(Automation)**: 복사 및 반복적 운영 작업 구성을 배제토록 내장 스크립트 인터페이스(`record-script`, `run-script`) 구조에 더해져 한정된 스크립트파일(`.velo`) 이행만으로 사람 없이 매크로 자동 통제가 가능도록 구성합니다.
